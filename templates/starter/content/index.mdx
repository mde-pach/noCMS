## How it works

Three plain ideas. Double-click any text on this page to rewrite it — what you see here is exactly what gets published.

<Grid columns={3} gap="md">
  <Card title="1 · Fork it">
    Start from this template. The whole site — content, theme, and the editor itself —
    is committed to **your** repo. Clone it, fork it, leave any time.
  </Card>
  <Card title="2 · Edit in place">
    Open the in-site editor, select a block to change its props, drag to reorder, and
    type directly on the page. Preview *is* the published page — one renderer, no surprises.
  </Card>
  <Card title="3 · Publish">
    One click commits to git and triggers a GitHub Action that builds static HTML to
    Pages. Versioned by git, hosted for free, served from the edge.
  </Card>
</Grid>

  </Container>
</Section>

## Theming is tokens, not code

Colors, type, spacing and radius live in a flat `theme.tokens` file and compile to CSS
variables at runtime. Change a token and the whole site restyles instantly — no rebuild,
no redeploy.

<Callout variant="tip">
  Open the editor and try the **Design & brand** panel. Change the brand color or body
  font and watch every block on this page update live.
</Callout>
